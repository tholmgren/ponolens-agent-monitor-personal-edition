Frequently asked questions

Understand what PonoLens sees, what it can stop, and how to use each protection safely.

01

Getting started

Install, connect, and test your agents.

What is PonoLens?

PonoLens is a local-first privacy monitor and Safe Prompt workspace for AI coding agents. It creates plain-language privacy receipts, detects configured sensitive information, and can stop supported prompts before submission.

How do I install and start PonoLens?

Run ./install.sh from the competition checkout. After the public installer is published, the documented one-line command will install the same local service. PonoLens opens http://127.0.0.1: in your browser and must remain running while agents are monitored.

How do I connect an AI agent?

Select Scan agents, enable the installed harness, then fully quit and reopen it so its hook configuration reloads. Use Test and submit a fictitious test prompt.

Which agents are supported?

are supported with different coverage. Claude Desktop is not Claude Code and is not supported. Detection alone does not mean the compatible hook is active.

Is PonoLens a native Mac application?

Not in the competition preview. The current edition is installed with install.sh, runs a local service, and opens its dashboard in a browser. A polished PonoLens Personal Mac application is planned after the competition.

How is PonoLens licensed?

PonoLens Personal Edition is open source under the Apache License 2.0. You may use, modify, and distribute it subject to the license terms. The license does not grant rights to the PonoLens name or trademarks.

Which operating systems does this beta support?

PonoLens Agent Monitor — Personal Edition (Beta) currently supports the Unix-based macOS environment only and creates a launcher in the user's Applications folder. A Windows release is planned for August 2026. Linux and other Unix platforms are not supported by this beta.

What do the harness status fields mean?
  • Installed: the application or CLI was found.
  • Hook configured: PonoLens configuration exists for that harness.
  • Currently reachable: PonoLens is actively receiving supported activity.
  • Prompt coverage: the prompt/actions exposed by that integration.
  • Capabilities: whether the integration supports Report, Redact, or Block when its hook runs.
  • Known limitations: activity the integration cannot reliably see or stop.
  • Last event received: the newest matching retained receipt.

Expand a harness accordion to see these details. Installed does not mean fully protected.

02

Protection modes

Know exactly what each Pono Guard option does.

Which protection mode is stable?

Report Only is the stable default. It lets the action continue and creates a redacted privacy receipt. A receipt cannot undo sensitive information that was already submitted.

What does Experimental Block do?

Block attempts to stop matching prompts at supported pre-submit hooks for . Coverage depends on the harness and hook. An event is red only when the integration actually stopped it; never treat the experimental setting as a universal prevention guarantee.

What does Experimental Redact do?

Redact detects sensitive information locally, creates a sanitized prompt, and attempts to stop the sensitive original at supported hooks. Review and resubmit the protected copy. Current third-party hooks do not provide one reliable cross-harness way to replace and submit composer text automatically.

Why are Redact and Block marked Experimental?

Agent harnesses expose different hooks, and those contracts can change. Experimental protection may miss activity or interrupt a supported workflow. Check each harness accordion for current prompt coverage, capabilities, known limitations, and the last event received. Use Safe Prompt whenever pre-submission interception is unavailable.

When does Pono Guard show Custom?

Custom · Advanced appears after category actions in Advanced Guard use a mixture that does not match the selected global Report Only, Block, or Redact mode. Selecting a standard mode applies that action to every category and removes the Custom state. Select Custom to return to Advanced Guard.

What is Experimental command monitoring?

It creates report-only receipts for commands exposed by supported agent hooks, such as shell, Git, package-manager, and network commands. It is off by default. PonoLens redacts detected secrets, credentials, personal information, and configured protected values before SQLite storage. A sensitive command is orange and appears under Needs Review. Command monitoring does not block or rewrite commands and does not monitor unrelated Terminal windows.

Can PonoLens protect Ollama requests?

Yes, when the calling application uses the PonoLens Ollama Gateway shown in Settings. Report Only forwards and records a redacted receipt. Block rejects matching requests before Ollama receives them. Redact removes configured protected values and forwards only non-streaming requests.

What are the Ollama Gateway limitations?
  • Direct calls to Ollama on port 11434 bypass PonoLens.
  • The MVP supports only /api/chat, /api/generate, and /api/tags, not Ollama's OpenAI-compatible, embeddings, model-management, or other APIs.
  • Report Only supports transparent streaming. A sensitive Redact request must use stream:false; streaming response re-identification is not supported.
  • The scanner covers textual chat content and generate prompt/system/suffix fields. It does not inspect images or every nested tool payload.
  • If PonoLens is stopped, applications configured only for its gateway cannot reach Ollama.
Do Block and Redact work in Codex side chats?

No. Codex side-chat prompts are not exposed early enough for reliable interception. PonoLens may observe them afterward. Use Safe Prompt before sending sensitive Codex content.

What do green, orange, and red mean?
  • Green: no configured sensitive information or unusual movement was detected.
  • Orange: sensitive information was detected and needs review.
  • Red: a supported adapter actually stopped the action.
Where do “Risks explained” and “Unsafe actions stopped” get their results?

The counts and review panels query the complete retained SQLite history. They do not filter only the 20 events shown in the home-page preview. Risks explained includes retained high- and critical-severity events; Unsafe actions stopped includes events whose recorded decision is Blocked.

03

Detection & Safe Prompt

Protect identifiers before they leave your device.

What sensitive information can PonoLens detect?

Categories include secrets, personal and contact information, healthcare, legal, financial, and organization-defined values. Examples include emails, identity documents, patient identifiers, matter numbers, validated cards and routing numbers, and common API-key formats.

Is sensitive-data detection perfect?

No. Deterministic detection can produce false positives or miss unusual formats. Validation and contextual patterns reduce noise, but important decisions still require human review.

What is Safe Prompt?

Safe Prompt scans locally, replaces identifiers with tokens, lets you review the protected draft, and sends or copies only the tokenized version. PonoLens can restore the original values locally after the response returns.

Does Safe Prompt send the original information?

The initial scan and token map stay local. Only the reviewed tokenized prompt is sent or copied. Always inspect the remaining narrative because context can identify someone even after obvious identifiers are removed.

Can I add company-specific identifiers?

Yes. Add exact values in Pono Guard. Advanced Guard also supports local dictionaries and constrained regular-expression rules for client codes, matter formats, project names, and internal classifications.

04

Data, privacy & compliance

Understand storage, retention, and legal boundaries.

Where are logs and settings stored?

The default database is ~/.ponolens/ponolens.db. Audit previews are redacted before storage. Policies and protected values stay in the local SQLite database; cloud synchronization is not enabled in this MVP.

How long are events kept?

The default is days. Change it in Settings to match your organization's approved retention and legal-hold policies.

What does the safe policy export include?

It exports a reusable policy template while excluding user-authored labels, exact protected values, dictionary entries, and custom regular-expression patterns. It is intentionally not a raw backup of sensitive local policy data.

What does “Delete all local data” remove?

After typed confirmation, PonoLens deletes local audit events, policies, protected values, dictionaries, regex rules, and local settings. It does not remove harness hook configuration or macOS Keychain credentials. This action is permanent.

What is included in CSV and PDF incident exports?

Exports contain redacted event metadata for the active Data Trail filters. Prompt content, detector samples, and raw protected values are excluded. Export files are still security records and should follow your organization's storage and sharing policy.

Does PonoLens establish HIPAA, legal, or SEC compliance?

No. PonoLens supports privacy and compliance workflows but does not itself establish compliance, preserve privilege, satisfy SEC obligations, or provide certification. Those outcomes require organizational, contractual, and technical safeguards beyond this tool.

Does deleting a sent prompt undo disclosure?

No. Deletion does not prove that transmitted data was never processed, retained, backed up, or logged. Treat it as cleanup, not reversal of disclosure.

What is a trusted destination?

A trusted hostname lowers the untrusted-destination risk score. It never disables sensitive-data detection or per-category Pono Guard actions.

05

Coverage & troubleshooting

Resolve common monitoring and integration questions.

What is MCP coverage?

PonoLens covers MCP calls explicitly routed through its gateway or exposed through a supported harness hook. It does not claim visibility into every MCP server or encrypted network request.

Which harness commands can PonoLens report?
  • Codex: commands found in covered task hooks and observable local session tool calls; unrelated Terminal commands are invisible.
  • Claude Code CLI: command tools exposed through PreToolUse; Claude Desktop is unsupported.
  • Cursor: command details exposed through its tool hooks; coverage and payload detail can vary by Cursor version and tool.
  • Windsurf/Devin: native pre-command events and compatible pre-tool activity.

Aliases, nested scripts, shell expansion, subprocesses, encrypted traffic, and commands that bypass the configured hook may hide the final operation or destination. Command output is not captured.

How does Data Trail filtering work?

Risk/activity, harness, date range, and redacted-metadata search can be combined. Filtering happens in SQLite before 100-event pagination, so it searches retained history rather than only the visible page. Daily summaries use the computer's local calendar day, and repeated-risk grouping uses matching retained results.

What happens if the PonoLens collector is unavailable?

Newly generated Cursor and Windsurf/Devin pre-submit bridges fail closed instead of silently allowing an unreviewed prompt. Reconnect an existing integration after upgrading to regenerate its bridge. Other coverage still depends on the behavior and hook contract of each harness.

Why is an installed agent missing from Scan agents?

The application or CLI may not be visible to the environment that started PonoLens. Restart that terminal or desktop app, confirm the executable is on PATH, and scan again.

Why is an agent detected but showing no activity?

Detection only proves the application or command exists. Confirm the integration is enabled, fully restart the harness, begin a new session, keep PonoLens running on the configured collector port, and submit a harmless test prompt.

Why does Claude Code say “Configured only”?

The hook configuration exists, but the Claude Code CLI is not visible to PonoLens. Confirm that claude is installed and on the launching environment's PATH. Claude Desktop does not satisfy this requirement.

How should I test PonoLens safely?

Use clearly fictitious names, emails, patient IDs, account numbers, and keys. Begin with stable Report Only, then test Experimental Redact and Block separately on a supported harness. Never use real regulated or authentication data merely to validate detection.
